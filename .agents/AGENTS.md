# Workspace Rules for AerialWindows

- **Git Version Control**:
  - Always stage and commit all modified files locally upon completing a coding task or bug fix.
  - Run `git push origin main` to automatically sync changes to the remote GitHub repository after committing.
  - Keep commit messages concise, descriptive, and prefixed with the scope of changes (e.g., `fix(proxy): ...` or `feat(settings): ...`).
